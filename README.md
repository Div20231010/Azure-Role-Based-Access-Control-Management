# Azure-Role-Based-Access-Control-Management
This project showcases a practical implementation of identity and access management using Microsoft Entra ID (formerly Azure Active Directory). The goal was to simulate a real-world scenario where a new user—Destiny Odianosen—is onboarded and assigned administrative privileges through a security group.
# 🔐 Azure Identity & Access Management Project

## Overview
This project demonstrates how to manage user identities and security groups in Microsoft Entra ID (Azure Active Directory) using the Azure portal.

## Objectives
- Create a user account for Destiny Odianosen.
- Set up a security group named "Senior Admins".
- Assign Joseph Price as both member and owner of the group.

##  Tasks Completed
### Task 1: Create User Account
- Created user: **Joseph Price**
- Username: `Destiny`
- Auto-generated password provided

### Task 2: Create Security Group
- Group name: **Senior Admins**
- Group type: Security
- Membership type: Assigned
- Member: Destiny Odianosen
- Owner: Destiny Odianosen

## Skills Demonstrated
- Identity lifecycle management
- Role-based access control (RBAC)
- Azure portal navigation
- Group and role configuration
